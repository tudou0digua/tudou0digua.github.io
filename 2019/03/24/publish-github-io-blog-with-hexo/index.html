3.3 编写

文章目录为test/source/_posts/，有个默认hello-world.md，可以对其进行修改，或者新建your_post.md，使用 Typora MarkDown编辑器进行编辑。

4.发布

安装hexo-server

安装hexo-deployer-git自动部署发布工具

生成静态页面

本地预览

浏览器中，输入http://localhost:4000，预览页面。

如果预览的页面是白板和Cannot GET / 几个字，终端定位到test目录,输入$ hexo list，对出现的Error XXXX is requried by XXX插件，使用$ npm install 插件名称 --save逐个安装所需的插件。

页面正常预览之后，发布页面到GitHub io

输入XXX/XXX.github.io，正常显示页面，表示发布成功。

如果，发布成功之后，页面修改内容没有及时更新，可能是浏览器缓存导致，可以换个浏览器打开试试。
